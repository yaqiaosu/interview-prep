# Promise 与异步

## 核心概念

Promise 是异步编程的一种解决方案，表示一个异步操作的最终结果。它有三种状态：
- `pending`：初始状态
- `fulfilled`：操作成功
- `rejected`：操作失败

状态一旦改变就不可逆（pending → fulfilled 或 pending → rejected）。

Promise 解决了回调地狱（callback hell）的问题，通过链式调用 `.then()` 让异步代码更可读。`async/await` 是 Promise 的语法糖，让异步代码看起来像同步代码。

## 常见追问

1. Promise 的链式调用原理是什么？
2. `Promise.all`、`Promise.race`、`Promise.allSettled`、`Promise.any` 的区别？
3. 如何手写一个符合 Promise/A+ 规范的 Promise？
4. 如何实现并发控制（比如最多同时发 5 个请求）？
5. `async/await` 的错误处理最佳实践？

## 回答要点

### 链式调用原理

`.then()` 返回一个新的 Promise，这个新 Promise 的状态由回调函数的返回值决定：
- 返回普通值 → 新 Promise 变为 fulfilled
- 返回 Promise → 新 Promise 跟随该 Promise 的状态
- 抛出异常 → 新 Promise 变为 rejected

### 静态方法对比

| 方法 | 行为 |
|------|------|
| `Promise.all` | 全部成功才成功，一个失败就失败 |
| `Promise.race` | 第一个 settled 的结果（无论成功失败） |
| `Promise.allSettled` | 等所有都 settled，返回每个的状态和值 |
| `Promise.any` | 第一个成功的结果，全部失败才失败 |

### 并发控制

核心思路：维护一个执行池，池子满了就等待，有任务完成就补充新任务。

```js
async function concurrentLimit(tasks, limit) {
  const results = [];
  const executing = new Set();
  for (const [i, task] of tasks.entries()) {
    const p = task().then(res => {
      executing.delete(p);
      return res;
    });
    results[i] = p;
    executing.add(p);
    if (executing.size >= limit) {
      await Promise.race(executing);
    }
  }
  return Promise.all(results);
}
```

### async/await 错误处理

- 用 `try/catch` 包裹 `await`
- 或者在调用处 `.catch()`：`await fetchData().catch(handleError)`
- 全局兜底：`window.addEventListener('unhandledrejection', handler)`
