# this 指向

## 核心概念

`this` 的值取决于函数的调用方式，而不是定义位置（箭头函数除外）。判断规则按优先级从高到低：

1. `new` 绑定：`this` 指向新创建的对象
2. 显式绑定：`call/apply/bind` 指定的对象
3. 隐式绑定：作为对象方法调用时，`this` 指向该对象
4. 默认绑定：独立调用时，严格模式下 `undefined`，非严格模式下 `window`

箭头函数没有自己的 `this`，它捕获定义时外层作用域的 `this`，且无法被 `call/apply/bind` 改变。

## 常见追问

1. 箭头函数和普通函数在 `this` 上的区别？
2. `call`、`apply`、`bind` 的区别？如何手写实现？
3. 类方法中 `this` 丢失的问题怎么解决？
4. React 类组件中为什么要 `bind(this)`？

## 回答要点

### 箭头函数 vs 普通函数

```js
const obj = {
  name: 'Tom',
  sayHi: function() { console.log(this.name); },    // 'Tom'（隐式绑定）
  sayHi2: () => { console.log(this.name); },         // undefined（捕获外层 this）
};
```

箭头函数适合用在回调中保持外层 `this`，不适合用作对象方法或构造函数。

### call / apply / bind

- `call(thisArg, arg1, arg2)`：立即调用，参数逐个传
- `apply(thisArg, [args])`：立即调用，参数以数组传
- `bind(thisArg, arg1)`：返回新函数，不立即调用

手写 `bind` 核心思路：
```js
Function.prototype.myBind = function(context, ...args) {
  const fn = this;
  return function(...newArgs) {
    return fn.apply(context, [...args, ...newArgs]);
  };
};
```

### 类方法 this 丢失

```js
class Button {
  constructor() { this.text = 'click'; }
  handleClick() { console.log(this.text); }
}
const btn = new Button();
const handler = btn.handleClick;
handler(); // this 丢失，undefined
```

解决方案：
- 构造函数中 `this.handleClick = this.handleClick.bind(this)`
- 使用类字段 + 箭头函数：`handleClick = () => { ... }`
- React 中也是同样的问题，函数组件 + Hooks 天然避免了这个问题
