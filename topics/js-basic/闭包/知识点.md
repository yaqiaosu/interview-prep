# 闭包

## 核心概念

闭包是指一个函数能够访问其词法作用域中的变量，即使该函数在其词法作用域之外执行。本质上，闭包 = 函数 + 其引用的外部变量。

JavaScript 引擎在函数创建时，会将函数体内引用到的外部变量"捕获"到一个内部的 `[[Environment]]` 记录中，这个记录不会因为外层函数执行完毕而被垃圾回收。

```js
function createCounter() {
  let count = 0;
  return () => ++count;
}
const counter = createCounter(); // createCounter 执行完毕，但 count 不会被回收
counter(); // 1
counter(); // 2
```

## 常见追问

1. 闭包会导致内存泄漏吗？什么场景下会？
2. 经典的 `for` 循环 + `setTimeout` 问题怎么解释？怎么解决？
3. 闭包和 IIFE 的关系是什么？
4. React Hooks 中的闭包陷阱是怎么回事？
5. 闭包在实际项目中有哪些应用场景？

## 回答要点

### 闭包与内存泄漏

闭包本身不会导致内存泄漏，但如果闭包引用了大对象且闭包本身长期存活（比如绑定在全局事件监听器上），那被引用的对象就无法被 GC 回收。解决方式是在不需要时手动解除引用或移除监听器。

### for 循环 + setTimeout

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0); // 输出 3, 3, 3
}
```

`var` 没有块级作用域，三个回调共享同一个 `i`，等到 setTimeout 执行时 `i` 已经是 3。解决方案：
- 用 `let` 替代 `var`（每次循环创建新的块级作用域）
- 用 IIFE 创建独立作用域：`(function(j) { setTimeout(() => console.log(j), 0) })(i)`

### React Hooks 闭包陷阱

函数组件每次渲染都会创建新的闭包，如果在 `useEffect` 或事件回调中引用了 state，拿到的是那次渲染时的快照值，而不是最新值。解决方式是用 `useRef` 保存最新值，或者正确设置 `useEffect` 的依赖数组。

### 实际应用场景

- 模块模式：用闭包实现私有变量
- 函数柯里化 / 偏函数
- 防抖节流函数中保存 timer 引用
- React Hooks 本身就是基于闭包实现的
