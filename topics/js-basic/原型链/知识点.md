# 原型链

## 核心概念

JavaScript 中每个对象都有一个内部属性 `[[Prototype]]`（可通过 `__proto__` 或 `Object.getPrototypeOf()` 访问），指向它的原型对象。当访问一个对象的属性时，如果对象自身没有，就沿着原型链向上查找，直到 `null` 为止。

关键关系：
- `构造函数.prototype` → 原型对象
- `实例.__proto__` → 原型对象（即 `构造函数.prototype`）
- `原型对象.constructor` → 构造函数

```js
function Person(name) { this.name = name; }
Person.prototype.sayHi = function() { return `Hi, ${this.name}`; };

const p = new Person('Tom');
p.sayHi(); // 沿原型链找到 Person.prototype.sayHi
p.__proto__ === Person.prototype; // true
Person.prototype.__proto__ === Object.prototype; // true
Object.prototype.__proto__ === null; // 原型链终点
```

## 常见追问

1. `new` 操作符具体做了什么？
2. `instanceof` 的原理是什么？
3. ES6 的 `class` 和原型链是什么关系？
4. 如何实现继承？几种方式的优缺点？
5. `Object.create(null)` 和 `{}` 有什么区别？

## 回答要点

### new 操作符

四步：
1. 创建一个空对象
2. 将空对象的 `__proto__` 指向构造函数的 `prototype`
3. 以该空对象为 `this` 执行构造函数
4. 如果构造函数返回了对象则用该对象，否则返回步骤 1 创建的对象

### instanceof 原理

沿着左侧对象的原型链向上查找，看是否能找到右侧构造函数的 `prototype`。所以 `instanceof` 不是检查"谁创建了谁"，而是检查原型链上的关系。

### class 与原型链

`class` 是语法糖，底层仍然是原型链。`class` 的 `extends` 实际上做了两件事：
1. 子类 `prototype.__proto__` 指向父类 `prototype`（实例方法继承）
2. 子类 `__proto__` 指向父类（静态方法继承）

### 继承方式对比

- 原型链继承：引用类型属性会被所有实例共享
- 构造函数继承（`call`）：无法继承原型上的方法
- 组合继承：调用两次父构造函数
- 寄生组合继承：最优方案，`Object.create(Parent.prototype)` 避免多余的父构造函数调用
- ES6 `class extends`：语法层面的寄生组合继承
