# 事件循环（Event Loop）

## 核心概念

JavaScript 是单线程语言，通过事件循环机制实现异步非阻塞。事件循环的核心流程：

1. 执行同步代码（调用栈）
2. 调用栈清空后，检查微任务队列，依次执行所有微任务
3. 微任务队列清空后，从宏任务队列取一个任务执行
4. 重复步骤 2-3

微任务（Microtask）：`Promise.then/catch/finally`、`MutationObserver`、`queueMicrotask`
宏任务（Macrotask）：`setTimeout`、`setInterval`、`requestAnimationFrame`、I/O、UI 渲染

关键点：每执行完一个宏任务，都会清空所有微任务，然后才执行下一个宏任务。

## 常见追问

1. `Promise` 和 `setTimeout` 的执行顺序？为什么？
2. `async/await` 在事件循环中是怎么执行的？
3. `requestAnimationFrame` 在事件循环中的位置？
4. Node.js 的事件循环和浏览器有什么区别？
5. 微任务中再产生微任务会怎样？

## 回答要点

### Promise vs setTimeout

```js
console.log(1);
setTimeout(() => console.log(2), 0);
Promise.resolve().then(() => console.log(3));
console.log(4);
// 输出：1, 4, 3, 2
```

同步代码先执行（1, 4），然后清空微任务队列（3），最后执行宏任务（2）。

### async/await

`await` 后面的表达式会立即执行，但 `await` 之后的代码相当于放在 `.then()` 回调中，属于微任务。

```js
async function foo() {
  console.log(1);
  await bar();       // bar() 立即执行
  console.log(2);    // 这行相当于 .then(() => console.log(2))
}
```

### requestAnimationFrame

rAF 的执行时机在微任务之后、下一次渲染之前。它不属于宏任务也不属于微任务，而是浏览器渲染流程的一部分。执行顺序：宏任务 → 微任务 → rAF → 渲染 → 下一个宏任务。

### Node.js 事件循环

Node.js 基于 libuv，事件循环分为 6 个阶段：timers → pending callbacks → idle/prepare → poll → check → close callbacks。`process.nextTick` 优先级高于 `Promise.then`，在每个阶段切换时都会清空。
