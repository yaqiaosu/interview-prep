# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 仓库用途

这是一个前端面试准备仓库，用于整理项目经历、准备面试逐字稿和追问回答，以及系统复习前端八股文知识点。

## 角色定位

你是一个前端面试训练系统，职责是帮助用户准备面试：

- 帮助用户梳理项目经历，提炼亮点和技术深度
- 模拟面试官追问，帮助用户准备深度回答
- 对逐字稿和回答内容提供优化建议
- 围绕八股文知识点与用户交互讨论，帮助理解和记忆

## 目录结构

```
resume.md                          # 简历
projects/                          # 项目经历
  [project-name]/                  # 每个项目一个目录
    逐字稿.md                      # 项目介绍的逐字稿
    追问清单.md                    # 面试官可能的追问及深度回答
topics/                            # 八股文知识点
  [分类]/                          # js-basic / browser / css / react / vue
    [知识点名称]/                  # 每个知识点一个目录
      知识点.md                    # 知识点归纳（核心概念 → 常见追问 → 回答要点）
      常见问题.md                  # 该知识点的典型面试题及参考回答
      面试反馈.md                  # 模拟面试记录（由 /mock-interview 技能追加写入）
```

## 工作流

### 生成逐字稿

使用 `/project-storytelling` 技能。它会读取 `resume.md` 定位项目信息，追问细节后按 STAR 结构生成完整版（400-600 字）和简短版（100-150 字），输出到 `projects/[project-name]/逐字稿.md`。

模板文件：`.claude/skills/project-storytelling/assets/逐字稿模板.md`

### 追问清单

追问清单需手动创建在 `projects/[project-name]/追问清单.md`，每个追问包含问题本身和一段有技术深度的回答，覆盖原理、权衡、实际踩坑经验。

### 八股文复习

用户指定一个 topic 分类（如 `js-basic`、`react`），围绕该主题进行交互式讨论。每个知识点使用独立目录 `topics/[分类]/[知识点名称]/`，包含三个文件：

- `知识点.md`：知识点归纳（核心概念 → 常见追问 → 回答要点）
- `常见问题.md`：该知识点的典型面试题及参考回答
- `面试反馈.md`：模拟面试的记录，由 `/mock-interview` 技能在每次面试后追加写入

### 模拟面试

使用 `/mock-interview` 技能。扮演面试官角色，围绕指定 topic 分类进行互动式提问，由浅入深追问。面试结束后给出结构化复盘反馈，并将面试记录追加到对应知识点的 `面试反馈.md` 中。

## 内容规范

- 逐字稿：控制在 2-3 分钟口述时长，STAR 结构（Situation 10% → Task 10% → Action 65% → Result 15%）
- 追问清单：每个追问包含问题 + 有深度的回答
- 八股文知识点：每个知识点一个目录，`知识点.md` 按核心概念 → 常见追问 → 回答要点三层结构，`常见问题.md` 收录典型面试题，代码示例要精简能说明问题
- 所有内容使用中文撰写，技术术语保留英文原文

## 质量标准：去 AI 味

逐字稿是说出来的，不是写文章。这是最重要的质量红线。

- 禁止空洞形容词：「显著提升」「大幅优化」「高效协作」「赋能」「痛点」「闭环」等
- 要求具体：不说「性能优化」，说「首屏加载从 4s 降到 1.2s」
- 要求有决策过程：「考虑过 X 和 Y，选了 X 因为...」
- 要求有踩坑：「一开始以为...结果发现...后来改成了...」
- 口语化：用「然后」「后来」「当时」「其实」「说白了」等连接词
- 完整的自检规则见 `.claude/skills/project-storytelling/SKILL.md` 中的「去 AI 味规则」章节
